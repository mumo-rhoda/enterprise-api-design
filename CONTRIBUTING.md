# Contributing to Enterprise API Design

Thank you for contributing! This guide explains how to add content consistently.

---

## Documentation standard

Every new section or topic MUST follow this exact structure:

```markdown
# Topic Title

## Introduction
## Why it matters
## Prerequisites
## Architecture
## Implementation
## Examples
## Best Practices
## Common Mistakes
## References
## Exercises
## Further Reading
```

## Adding a new section

1. Create a folder: `/<section-name>/`
2. Add `README.md` following the doc standard above
3. Add examples in `/examples/<section-name>/`
4. Add diagrams to `/diagrams/` with descriptive filenames
5. Update the root `README.md` table

## Code examples

- Use real, runnable code or spec — no pseudocode
- Include comments explaining the *why*, not just the *what*
- Provide both a "bad" and "good" example where applicable

## Pull request checklist

- [ ] Follows the documentation standard
- [ ] Includes at least one working example
- [ ] Includes at least 3 exercises
- [ ] References link to official sources
- [ ] Spelling and grammar checked
